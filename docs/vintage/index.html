Vintage

#Vintage

Vintage is a human-friendly label attached to a version to time series saved in the database. When one or many time series are saved together in a collection, a vintage is automatically created. It comes really handy:

  • when you want to share the version of your data and refer to it in a human-friendly label.
  • when you want to save metadata with a particular version of the series

#Sharing the right version of your data

Vintage name is a human-readable label assigned to a timestamp of when data was written to the collection. Therefore, it allows you to refer in an easy way to a particular version of data.

For example, if you save revenue projections for a review by the management, you can send them a vintage name and an HTML link to this particular version of data.

#Metadata

Vintage can be very useful when you want to save some metadata (JSON object)) about one or many time series you want to save in the database.

For example, suppose you have a model that takes some parameters as input and generates output, which is a bunch of time series. You can save the output to a collection and attach the parameters of the model using a vintage. This will allow you to reproduce the output of your model if required.

You could also store some extra statistics in a vintage. This could be handy when someone comes with questions regarding the output of your model.

You can also store some attributes, which will permit you to search for the right version of the time series.

#What you need to know is

  • Vintages are unique within a collection and can only refer to the data saved in that particular collection.
  • Vintage is created every time data are saved.
  • Vintage's name, description, attributes can be edited after it was created when time-series were written to a collection.
  • Vintages are fully-versioned and you can see the historical modifications to any vintage.
  • Vintages cannot be deleted.
  • Metadata can by any JSON object by to 15MB.
