Time series data

#Saving data

There are three methods of saving time series data: 'update', 'append', 'overwrite'. Each represents a different way existing data is modified with the data being saved.

In Python, a method of saving data can be selected using the 'method' parameter in the 'save' method of Collection or Timeseries objects. The default method is 'update'.

ts.save(pd.Series(np.array([10,20,30,40,50,60]), index=pd.period_range(start=pd.Period("2000Q1"), periods=6)), method='update')

#Vintages

A vintage is an object referencing a specific version of a time series at the given realtime. A vintage is created every time a single or multiple series are saved in the database. Vintage is uniquely identified by its name. The description field in a vintage object contains more detailed description of the specific data version. Metadata allows one to save JSONifiable metadata pertaining to a specific data version. For example, when a collection contains a forecast generated using some model, metadata can contain model version as well as parameters used to produce a given forecast.

forecast = pd.DataFrame({
    "x": pd.Series(np.random.rand(6), index=pd.date_range(start=pd.Timestamp("2000-1-7"), periods=6, freq="D")),
    "y": pd.Series(np.random.rand(6), index=pd.date_range(start=pd.Timestamp("2000-1-9"), periods=6, freq="D"))
}) 
coll.save(forecast, 
          vintage=pychronos.Vintage(
            name="2000Q1 release",
            description="Division X financials",
            metadata={'model': '1.2.2', 'parameters': {'alpha': 123 }}
          )
        )
